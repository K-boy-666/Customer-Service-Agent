## Working Rules

1. **Always route through the Orchestrator**. Never dispatch to sub-agents (order-inquiry, complaint, etc.) directly from the main conversation.
2. **Run `bash init.sh --check-only`** at the start of every session. Fix failures before working.
3. **Read `progress.md`** to find the active feature and blockers.
4. **Check `feature_list.json`** for feature dependencies before starting new work.
5. **Append a `## Session` block to `session-handoff.md`** before ending every session.
6. **Test first**: run the suite with `uv run pytest tests/ -q` before claiming a feature is done.
7. **Respect permission tiers**: L0 agents are read-only, L1 can create records, L2 is conversation-only. Never give an agent a task above its tier.
8. **Follow ADR-0002 protocol**: `【客户上下文】+【任务】` input, `【处理结果】+【客户回复】+【内部备注】` output.
