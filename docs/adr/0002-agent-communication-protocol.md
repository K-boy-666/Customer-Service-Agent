# ADR-0002: 标准化子 Agent 通信协议

## 状态

已采纳 (2026-06-11)

## 背景

Orchestrator 需要调度 6 个子 Agent，每个子 Agent 由不同开发者编写。如果没有统一的输入/输出格式，Orchestrator 整合结果时需要针对每个子 Agent 做格式适配，维护成本随子 Agent 数量线性增长。

## 决策

采用**结构化自然语言协议**而非纯 JSON 或纯自由文本：

- **输入**：`【客户上下文】` + `【任务】` 两个语义区块
- **输出**：`【处理结果】`（状态码）+ `【客户回复】`（自然语言）+ `【内部备注】`（元信息）

## 理由

1. **纯 JSON 对 LLM 不友好**：模型在生成结构化 JSON 时容易出错（缺括号、字段名拼错），自然语言区块配固定标签更鲁棒。
2. **纯自由文本对 Orchestrator 不友好**：无法可靠区分"给客户看的回复"和"给 Orchestrator 的内部备注"。
3. **区块标签方案折中**：用固定标签（`【客户上下文】`、`【客户回复】`）提供结构锚点，用自然语言填充内容，兼顾解析可靠性和生成鲁棒性。

## 后果

- 所有子 Agent 的 system prompt 必须包含输出格式规范
- Orchestrator 整合结果时按 `【客户回复】` 标签提取并拼接
- 新增子 Agent 时必须遵循同一协议
